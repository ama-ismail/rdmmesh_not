# rdmmesh — docs

| Документ | Назначение |
|---|---|
| [`../SPEC.md`](../SPEC.md) | Полная архитектурная спецификация (постановка, бизнес-процессы, ТЗ, дорожная карта). Главный документ. |
| [`adr/`](./adr/) | Architecture Decision Records — фиксированные точки решений с обоснованиями и trade-off'ами. |
| [`handoff/`](./handoff/) | Передаточные документы между сессиями / агентами. Самодостаточные снимки состояния — сюда смотрит агент, который только что подключился к проекту. |

## Что куда писать

- **SPEC.md** — обновляем при изменении scope, ролей, контрактов API, тех-стека.
- **ADR** — новый файл при каждом значимом архитектурном решении (`docs/adr/NNNN-short-title.md`).
- **Runbooks** для оператора, инструкции для пользователей — добавятся по мере приближения к продуктиву (см. SPEC §5.4 Definition of Done).
