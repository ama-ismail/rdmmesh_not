## 0. TL;DR

- **Threat model.** RDM_ADMIN деплоит per-domain BPMN с
  `<rdm:workflowGraph>`. SPEC §3.2 #7: даже RDM_ADMIN не вправе обойти
  4-eyes. Цель ревью: может ли (ошибочный/over-privileged/скомпром.)
  admin крафт-графом ослабить self-approval/no-bypass для домена.
- **F-B1 (High, OWASP A01/A04) — ИСПРАВЛЕНО.** Сеть B2 доказывала
  «STEWARD-ребро затем OWNER-ребро», но OWNER-guard ловит steward==owner
  только через `actor ∉ reviewers`, а reviewers заполняется лишь если у
  STEWARD-ребра `recordReviewer=true`. Граф со STEWARD-ребром
  `recordReviewer=false` проходил сеть → один человек = steward И owner
  (4-eyes → 2-eyes). **Фикс:** инвариант — каждое non-reject
  STEWARD-ребро ОБЯЗАНО `recordReviewer=true`.
- **F-B4 (Medium, OWASP A04) — ИСПРАВЛЕНО.** SYSTEM-рёбра были
  неограничены — крафт-граф мог описать `DRAFT→PUBLISHED` (SYSTEM).
  Эксплойт-порог высокий (SYSTEM требует RDM_SYSTEM, его нет у людей),
  но граф не должен и описывать иной системный маршрут. **Фикс:**
  SYSTEM-ребро допустимо только формой `OWNER_APPROVED→PUBLISHED` /
  `PUBLISHED→DEPRECATED`.
- **Integrity — ИСПРАВЛЕНО.** Ребро в `OWNER_APPROVED` теперь обязано
  `setApprover=true` (иначе `approved_by`/HMAC-подпись E6 не
  фиксируются — нарушение аудита, не bypass, но integrity-gap).
- **Подтверждённые сильные места:** runtime fail-safe → дефолт +
  re-validate инвариантов на чтении (tampered DB-row не ослабит
  no-bypass); deploy-gate до Flowable; SUBMIT/SYSTEM не ведут в
  терминал; reject только STEWARD/OWNER; codec — enum-strict, тело
  капнуто F3 (E14.15). Граф-пространство ограничено 7 enum-статусами →
  перебор путей не DoS.

## 4. Остаточный риск / ограничения

1. **Доказательство — структурное, не формально-верифицированное.**
   `WorkflowGraphInvariants` перебирает простые пути на 7-узловом
   пространстве статусов (исчерпывающе для текущего enum). При
   расширении `Status`/`Kind` — пересмотреть теорему и тесты.
2. **RDM_ADMIN-доверие.** Deploy — `@RolesAllowed("RDM_ADMIN")`. Ревью
   снижает риск *ошибки/злоупотребления* конфигурацией, но компрометация
   RDM_ADMIN-учётки — отдельный контур (Keycloak/AD, вне scope B3).
3. **Нет UI-редактора графа** — граф пишется в BPMN-файле руками;
   ошибки ловит deploy-gate (→400). UI/дифф графа — V1+.
4. **prod-gate.** B функционально готов и compliance-safe (двойной
   рубеж + ревью). Перед массовым prod-включением кастомных графов —
   подтвердить findings-fix на CI (ниже) и согласовать с банковским
   security/compliance (как E14.8 закрывал DoD §5.4 — здесь аналогично
   для B).
